How Python Code Gets Executed

A Python 'Source Code', is executed line-by-line, from the top to the end of the code, in two sequencial steps:

  • Compilation Step:
    Is the first step and is where Python 'Source Code' is compiled into an intermediate code, machine code, known as 'Byte Code', creating a file that ends with the suffix '.pyc'.
  • Execution Step:
    Is the second step and is where the 'Byte Code' gets executed by a PVM ('Pyton Virtual Machine') and is translate into 'Native Code'.

'Native Code' will then allow programs to run on a particular operating system (Windows, Mac and Linus).

Note that '.py' files contain 'Source Code', whereas '.pyc' files contain 'Byte Code'.

Whenever a Python program is called, Python will check, if a compiled version with the '.pyc' suffix exists. This file has to be newer than the file with the '.py' suffix. If such a file exists, Python will load the 'Byte Code', which will speed up the start up time of the script. If there is no 'Byte Code' version, Python will create the 'Byte Code' before it starts the execution of the program.

How does Python interact with other PL

Python is an advanced scripting language that is being used successfully to glue together large software components. It spans multiple platforms, middleware products, and application domains.

Is easily extensible with C/C++/Java code, easily embeddable in applications and supports object-oriented programming. Python programmers can create classes, use multiple inheritance, define methods, overload operators, and so on.

Python is in use at many places as an integration language, used to glue together ("steer") existing components. The strategy here is to create Python extension modules (written in C/C++) that make the functionality of large components written in C/C++ available to the Python programmer.

The extension ("glue") modules are required, because Python cannot call C/C++ functions directly, the glue extensions handle conversion between Python data types and C/C++ data types and error checking, translation error return values into Python exception.

Creation of glue extensions is simplified by the existence of SWIG, which reads header files containing function and method prototypes and automatically generates the necessary type conversion and error checking code. In situations where the underlying code (usually C code) doe not use an object-oriented model, the glue extension can in turn be wrapped in a Python module that defines a proper class hierarchy, while delegating the performance critical operations to the C code.

Because Python has existing interfaces to so many different components in very different application domains, Python is ideal for oddball integration tasks. It can link a commercial database to number-crunching code; it can add a graphical user interface to a network management tool; it can send email from a virtual reality application, and so on.

Variables

Variables are used to temporarily store data in a computer's memory. For example:

            price = 10

When Python interpreter execute this code, it will allocate some memory, then it will store the number '10' in that memory, and finally it will attach the 'price' lable in that memory location.

As a metafor image we have a box, that we put the number '10' inside that box, and then put the label 'age' on the box. Now, it is possible to use this label, anywhere in our program to access the value that is inside that box. To see the result, just print it on the terminal:

            price = 10
            print(price)

In a more technical descriptition of the process, the storage of variables works like this:

  • When number '10' is about to be store in the memory, first it will be converted to it's binary representation, which will be a bunch of 0's and 1's, and only after it will be stored in the computer's memory.
  • Remember that Python is completely object oriented, and not statically typed, so the value of the variable, it will be the last value assign to that variable, beacuse Python interpreter read the code line-by-line, from the top to botton. In the example bellow, the variable 'price' will be '20', and not '10', because the 'price' was set to '10', but then it was reset to '20' (last value assigned):
    •                 price = 10
                price = 20
                print(price)

In resume, to define a variable:

  • Start by defining an 'Identifier' (in this case: 'price'), which is the name of the variable.
  • After, assign a value to the variable (in this case: '10'), using the assign operator equal (' = ').
  • It should always be used, lower cases letters, remember that Python is a case sensitive language.
  • If a variable name has multiple words in the name, it should be used an underscore ' _ ' to separate those words, for example, 'is_published'.
  • Variables should always have descriptive names.

Sometimes there is also a need to assign the same value to several variables, to avoid making individual assignments:

            price_a = price_b = price_c = 100

Another particularly useful thing are temporary variables. They are used to change values between variables. In the following example, 'temp' represents a temporary variable:

            product_1 = '10 Units'
            product_2 = 'Out of Stock'
            
            temp = product_1
            product_1 = product_2
            product_2 = temp

As a result we will obtain 'product_1 = 'Out of Stock' ' and 'product_2 = '10 Units' '. We changed we values of each variable.

Strings

Strings ('str') in Python are defined using single quotes (' ') or double quotes (" "). To define a multi-line string, we surround our string widh triple quotes (" " "). Example:

In the case above, is necessary to use double quotes, because we want to wrap the word Beginners inside double quotes. For Python, and without the double quotes, the 'str' would end in "Python Course for" and it would not be able to know what the rest of the characters ("Beginners") represented.

If we want a 'srt' that expandes throughout multiple-lines, we can use 3 quotes, like this:

            course = '''
            Hello Paulo,
            
            Here is our first e-mail to you.
            
            Thank you,
            The company manager
            '''
            
            print(course)

We can get an individual character in a 'str', using square brackets '[ ]' and specifing the index of that character:

            course = 'Python for Beginners'
            print(course[0])

In this case the result will be 'P', because the index '0' of this 'str' is 'P'.

How Python index 'str' are set:

            'Python for Beginners'
            '01234567...'

It is also possible to use negative index. In these cases, Python will start counting from the end of the str', to the start of the 'str':

            course = 'Python for Beginners'
            print(course[-1])

In this case, the result will be 's'. How Python 'str' are index, using negative index:

            'Python for Beginners'
            '           ... -2 -1'

Notes:

  • 'course[0]': returns the first character.
  • 'course[1]': returns the second character.
  • 'course[-1]': returns the first character counting from the end.
  • 'course[-2]': returns the second character counting from the end.

We can also slice a 'str', using a similar notation:

            print(course[1:5])

In this case the result will be 'ython'. Note that the above expression returns all the characters starting from index position of '1' to '5', but excluding '5'.

Notes:

  • If we leave out the start index, '0' will be assumed.
  • If we leave out the end index, the length of the 'str' will be assumed.
  • If we do not set any values ( 'course[ : ]' ), the whole 'str' will be returned.

Is also possible to use formatted 'str' to dynamically insert values into our 'str':

And to check if a 'str' contains a character (or a sequence of characters), we can use the 'in' operator: 

            course = 'Python for Beginners'
            contains = 'Python' in course
            print(contains)

The return result will be 'True' (boolean value).

While loops

'While loops' are used to execute a block of code, multiple times. They are often used to build interactive programs and games.

We write 'while statements' and right after, we type the condition followed by a colon. As long as the condition is true, the code will be repeatedly executed. When the condition is false, the cycle ends. Here is an example:

            i = 1
            
            while i <= 5:
                print(i)
                i = i + 1

In this example, the variable 'i' has set to '1', and as long as 'i' is less or equal to '5', the result will be printed on the terminal. We use 'i = i + 1', to increment 'i' by '1', after each cycle, and the reason we do this is to ensure that the 'while loop' will end, otherwise 'i' would be '1' forever, so we would end with an infinite loop, because the condition would always be true ('i = 1' is always less than '5').

Let us see a more complex example. Imagine we want to build a guessing game, using a 'while loop'. Suppose we have to guess what is the secret number of the game and that we have 3 attempts to guess it. If we do not guess, we will receive a message saying 'Sorry you failed!' and if we guess it, a message saying 'You win!'. Code could be something like this:

            secret_number = 9
            guess_count = 0
            guess_limit = 3
            
            while guess_count < guess_limit:
                guess = int(input('Guess: '))
                guess_count += 1
                if guess == secret_number:
                    print('You won!')
                    break
            
            else:
                print('Sorry you failed! Try again!')

Notes:

  • 'while loops' and 'if statements' can optionally have an 'else' part, or they can be terminated by using the 'break' statement. In this case, 'if statement' does not have an 'else' part, but has a 'break' statement. When Python interpreter sees this, it is going to immediately terminate our loop, it is not going to evaluate the condition ( 'guess_count < guess_limit' ) again.
  • The 'else' part in the 'while loop' will get executed, if the 'while loop' completes successfully the 3 attempts, without an immediate 'break'.
  • In resume, if the users guess the number, in one of the 3 attempts, Python interpreter will break the loop and jump out of it, so the code in the 'else' block will not be executed. But if the user can not guess the number in the 3 attempts, Python interpreter will never break out of the loop, and it will run until the 'while loop' is completed (3 attempts) and the condition set ('guess_count < guess_limit') becames false. In that case, the code in the 'else' block will get executed.

Nested Loops

'Nested loops' basically means adding one 'loop' inside of another 'loop', and with this technique we can do some amazing things. As an example we can generate a list of coordinates, with 'x' and 'y' values.

Lets imagine we want to generate all of this coordinates: (0, 0), (0, 1), (0, 2), (1, 0), ... (3, 3).

            for x in range(4):
                for y in range(4):
                print(f'({x}, {y})')

In the first iteration and in line one of the code, we get 'x = 0', and in line two (in the 'inner loop') and also in the first iteration we get 'y = 0 ', so it will be printed on the terminal '(0, 0)'.

Now the controls go back to line two, or our 'inner loop' and on this second iteration 'y = 1', but we are still in the first iteration of our 'outer loop', so 'x = 0', but now 'y' is incremented to '1'. That is why we see '(0, 1)' on the terminal.

This process will continue until the 'inner loop' ('for y in range(4)'), is completes. That is when 'y' reaches '3', because the range function of the 'inner loop' generates numbers between '0' to '4', but not including '4', so we will have '0', '1', '2' and '3'.

After this 'inner loop' completes, then the control goes back to line one, and at this point we are going to be in the second iteration of our 'outer loop'. So 'x' will be '1', then the control will be moved to line two, or our 'inner loop', at this point this range function is going to generate the numbers '0', '1', '2' and '3', so the 'inner loop' will be executed once agian for 4 times, and then it will go back to the 'outer loop'. This is how 'nested loops' get executed.

Lists

In Python we can define 'lists' using square brackets '[ ]'. We can have a list of items, a list of numbers, a list of costumers, a list of emails, products, whatever. Let us see an example:

            names = ['John', 'Paulo', 'Eli', 'Sarah', 'Mary']
            print(names[1])

In those 'lists' we can access each individual item, just like we did to access each character of a string, by using the index number of the item we want. In the example above, we used index '1', so we will see printed on the terminal, the item of the list 'names' with the index '1' that is 'Paulo'.

To access the item of a 'list', we can use positive index, negative index and a range, to specify the data we want to extract from the 'list', and the 'original list' is not affected, it stays the same.

Let us see some examples of ways to extract items from 'lists':

  • 'course[0]': returns the first item.
  • 'course[1]': returns the second item.
  • 'course[-1]': returns the first item counting from the end.
  • 'course[-2]': returns the second item counting from the end.
  • 'course[1:]': returns a list with all the items, starting on item with index '1', in this case 'Paulo'.
  • 'course[:3]': returns a list with all the items, ending on item with index '2', in this case'Eli', and excluding item '3', in this case 'Sarah'.
  • 'course[:]': returns a similar list as the original one.

The items of a 'list', can also be easilly modified, for example if we made a mistake. For that we should access the item we want to change, using the index item number and set it to a new value. Suppose we want to change the first item of the 'list' from 'John' to 'Jon':

            names[0] = 'Jon'
            print(names)

Now the 'list' will be '['Jon', 'Paulo', 'Eli', 'Sarah', 'Mary']'. By doing this, we update the 'list', we overwrite it, and we no longer have the 'original list'.

Let us see another example, by writing a program to find the largest number in a 'list':

            numbers = [3, 6, 2, 8, 10, 4]
            max = numbers[0]
            
            for number in numbers:
                if number > max:
                    max = number
            
            print(max)

In the example above, we assume that the maximun number in our 'list', is the first item, in this case '3'. Chances are our assumption is wrong, so we need to iterate over the list, we need to loop through it, get each item and compare it with 'max'. If it is greater than 'max', then we need to reset 'max' to that number.

So in the first iteration we get '3' and the 'max' is also '3'. Is '3' greater than '3'? No. So we move on, we get the second number, is '6' greater than '3'? It is. So we need to reset 'max' to '6'. Once again, we continue, we get '2'. Is '2' greater than '6'? No. So we move on. And we continue like this, until we check all the item of the 'list'.

2D Lists

'Two dimensional lists' are extremely powerfull and they have a lot of applications in data science and machine learning.

For example, in math we have a concept called 'matrix', which is like a rectangular array of numbers, where we have rows and columns. We can model this in Python, using '2D lists'. A '2D lists' is a list, where each item in that list is another list. Let us see an example:

            matrix = [
                [1, 2, 3],
                [4, 5, 6],
                [7, 8, 9]
            ]

In the above example, we defined a 'matrix' and we set it to a list. Each item in this list, is going to be another list, and that list represents the items in each row.

So the first item in our list, is going to be another list, and in the first list we are going to have the values '1', '2' and '3' ('[1, 2, 3]'). Now, the second item in our matrix list, once again, will bea list. This list represents the items in the second row, so the values '4', '5' and '6' ('[1, 2, 3]'). And finally the third list, items '7', '8' and '9' ('[7, 8, 9]'). Note that we separate each list in each row by using a ','.

So as you can see, we have a two dimensional list. Each item in our outer list, is another list, and this is how we can implement a 'matrix' in Python.

Now, to access an individual item in our 'matrix', once again we use square brackets '[ ]'. Lets suppose we want to access the number '2' in our 'matrix':

            print(matrix[0][1])

In this code, first we need to go and get the first item in this list, so we pass '0'. Now this expression returns another list. That is the inner list. In this list, let's say we want to access the second item, so once again, we add another square brackets, and we pass '1', that is the index of number '2' in this list. So, by using two square brackets, we can access individual items in our 'matrix'.

It is also possible to modify items in our 'matrix', like this:

            matrix[0][1] = 20
            print(matrix[0][1])

Or use 'nested loops' to iterate over all the item in this 'matrix':

            for row in matrix:
                for item in row:
                    print(item)

In the code above, we start with our 'rows', so in that loop ('for row in matrix:') we are iterating over our 'matrix' list, and in each iteration, 'row' will contain one item, in this case the item is a list (for example '[1, 2, 3]'). And after, we use an 'inner loop' ('for item in row:'). The final result in this case, is all the items of our 'matrix' ('1', '2', ..., '9').

List Methods

In this section, we will explore some list 'methods' or 'functions', these are the operations we can perform in a list, like adding items, removing existing items, check for the existence of an item, and so on. To access these 'methods' we use the 'dot' notation. Let us define a list of number and execute some of these operations:

            numbers = [5, 2, 1, 5, 7, 4]
            numbers.append(20)
            
            print(numbers)

In this code, we are adding the number '20' to our list. When we print the list on the terminal, this new number, will appear on the end of that list.

Let us see some 'methods' we can use with lists:

  • '.append(20)': add a new item to a list. The new item will appear on the end of the list.
  • '.insert(2, 10)': add a number in the position we want (middle, beginning, ...). This 'method' takes two values, the first represents the 'index' at which we want to insert the new value (in this case '2'), and the second represents the actual 'object' we want to add to the list (in this case '10').
  • '.remove(20)': remove an item from a list.
  • '.clear()': remove all the item from a list. This 'method' does not take any values.
  • '.pop()': remove the last item from a list. This 'method' does not take any values.
  • '.count(5)': count the occurance of an item (in this case return '2', because we have two '5').
  • '.sort()': sort the list in an ascending order. This 'method' does not take any values.
  • '.reverse()': reverse the items in a list. The 'reverse' method, can be used, for example, after the '.sort()' method, sorting the items in descending order.
  • '.copy()': copy the list. This 'method' does not take any values, and the new list will not be updated if we make changes on the inicial list, because this are two independent lists.
  • '.index(5)': check the existence of an item in a list. This 'method' returns the first occurence of this item. If it is passed a value that it is not on the list, we will obtain an error message.
  • 'print(50 in numbers)': check the existence of a character, or a sequence of characters, in a string, using the 'in' operator with a string. In this case because '50' is not in the list, we will obtain a boolean value, in this case 'False'. Does not generate an error message.

Let us see an example of the utility of these 'methods', by writing a program that allow us to remove all the duplicate itens in a list:

            numbers = [2, 2, 4, 6, 3, 4, 6, 1]
            uniques=[]
            
            for number in numbers:
                if number not in uniques:
                    uniques.append(number)
                
            print(uniques)

In the code above, first we need to define a new list ('uniques') and set it initially to an empty list. Then we need to iterate over our first list ('numbers'), get each item, and if we do not have that number in the 'unique' list, then we will add it to this second list.

Tuples

'Tuples' are another important structure in Python. 'Tuples' are similar to lists, so we can use them to store a list of items. But unlike lists, we can not modify them, we can not add new items, we can not remove existing items. We say 'tuples' are immutable. We can not mutate or change them.

Most of the times we use lists in our programs, but 'tuples' are also useful. For example, to create a list of items and we want to make sure that accidentally we do not modify that list, then it is better to use a 'tuple'.

To define 'tuples' we use parentheses '( )':

            numbers = (1, 2, 3)

And to access the 'methods' available for 'tuples', we use the 'dot' notation. Note that are several 'methods' that were available to 'list', but they are not available for 'tuples', like '.append()', '.insert()', '.remove()', '.clear()' and '.pop()', so we can not add or remove any of the items in a 'tuple'.

We can not change a 'tuple', we can only get information about it, using one of the two 'methods':

  • '.count(self, x)': use to count the number of occurrences in an item.
  • '.index(self, x, start, end)': use to find the index of the first occurrences of an item.

Note that using the 'dot' notation to access the 'methods', we can also see other 'methods' that start with two underscores ('__'), we refer to these as 'magic methods'.

Similar to lists, we can access individual items, using square brackets. As an example, we can get the first item like this:

            print(numbers[1])

As final notes, if we try to assing a new item to a 'tuple', using in this case 'numbers[0] = 10', we will obtain an error message saying ' 'tuple' object does not support item assignment '.

Unpacking

'Unpacking' is one of the powerful features that Python has. Let us suppose we have this 'tuple':

            coordinates = (1, 2, 3)

Imagine that '(1, 2, 3)', represents the coordinates '(x, y, z)'. Now lets say we want to use this values in a few complex expressions in our program. Maybe we want to include them as part of a large complex formula. So we would have to write code like this:

            result = coordinates[0] * coordinates[1] * coordinates[2]

As we can see in this sort example, our code is getting to long. But let us say we want to use these values in quite a few places in our program, so a better approach is to get these values and store them in separate variables, that we can get, like this:

            x = coordinates[0]
            y = coordinates[1]
            z = coordinates[2]
            
            result = x * y * z

But in Python we can achieve the same result with far less code. So we can define our variables and set them to a 'tuple':

            coordinates = (1, 2, 3)
            x, y, z = coordinates

When Python interpreter see the code above, it will get the first item of the 'tuple' ('1'), and assign it to the first variable ('x'). Then it will get the second item of the 'tuple' ('2'), and assign it to the second variable ('y'). And similarly, it will get the third item of the 'tuple' ('3'), and assign it to the third variable ('z'). So, we are 'unpaking' this 'tuple' into 3 variables 'x', 'y' and 'z'. So if we print each 'x', 'y' or 'z', we will see '1', '2' or '3' in the terminal.

'Unpacking' is not limited to 'tuples', we can also use this feature for lists as well:

            coordinates = [1, 2, 3]
            x, y, z = coordinates

In this code, we changed '( )' to '[ ]', and by doing this, we have change 'coordinates' from a 'tuple' to a 'list', so we can also 'unpack' our list into 3 variables as we did it for 'tuples'.

Dictionaries

We use 'dictionaries' in situation where we want to store information that comes as a 'key value pairs'.

Let think in a customer, for example. A customer has a bunch of attributes like name, email, phone number, address and so on. Each of these attributes has a value:

  • Name: John Smith
  • Email: john@gmail.com
  • Phone: 1234

This is what we call 'key value pairs', where 'Name', 'Email' and 'Phone' are the 'keys' and each 'key' is associated with a value, 'John Smith', 'john@gmail.com' and '1234'.

So this is where we use a 'dictionary'. With a 'dictionary' we can store a bunch of 'key value pairs'. So let us see, how we can define a dictionary in Python.

First we define a variable 'customer' and we set it to curly braces '{ }'. With these curly braces, we can define a 'dictionary', by seting 'key value pairs', inside the curly braces:

            customer = {
                'name': 'John Smith',
                'age': 40,
                'is_verified': True
            }

Note that:

  • Each 'key value pairs' is separated with a ','.
  • 'Keys' should be unique in a 'dictionary', just like in a real dictionary, where each word is only listed once. It is not allowed to have two or more 'keys' with the same name.
  • 'keys' can be strings or numbers.
  • The 'values' of the 'keys' can be anything, a string, a number, a boolean, a list, and so on.

Now we can access each item of this 'dictionary' using square brackets, like this:

            print(customer['name'])

In this code, we specified a 'key' and this returned the value associated with that 'key', in this case, it returned 'John Smith'.

Note that, if we specify a 'key', for example 'birthdate', that has no associated value to it, we will get an error message, because we do not have a 'key' called 'birthdate'. We will also get an error message if we use un upper case to call a 'key', that is not define with a capital letter, for expample 'Name'. This happens, because Python is a case sensitive language.

To avoid getting an error message, we can use the 'get method':

            print(customer.get('name'))

Using the 'get' method, we assure that we receive the 'value' associated with the 'key', in this case 'John Smith', or if the 'key' does not exist, we receive 'None' on the 'Terminal', but we will never receive an error message. Remember that 'None' is an 'object' that represents the absence of a value. We can also supply a 'default value' in case the 'dictionary' does not have the 'key' we are asking, like this:

            print(customer.get('birthdate', 'Jan 16, 1975'))

With theis line of code, instead of getting 'None' on the terminal, because 'birthdate' does not exist, we will get 'Jan 16, 1975'.

Similarly with lists, we can also update a 'dictionary' value:

            customer['name'] = 'Jack Smith'
            print(customer['name'])

Or we can add a new 'key' to a 'dictionary':

            customer['birthdate'] = 'Jan 16, 1975'
            print(customer['birthdate'])

Let us see a more complex example, where a program asks for our phone number and then it take those numbers and translate then to words:

            phone = input('Phone: ')
            digits_mapping = {
                '1': 'one',
                '2': 'two',
                '3': 'three',
                '4': 'four'
            }
            
            output = ''
            
            for number in phone:
                output += digits_mapping.get(number, '!') + ''
            print(output)

On this code, first we get the user phone number and store it on a variable called 'phone'. After we need to check each charater that the user has entered, and translate that character using a 'dictionary', because a 'dictionary' is a struture that allow us to map a 'key' to a 'value'.

Now we need to loop through the phone string. For that we can use '[ ]' or the 'get' method. By using the 'get' method, we can define an error message, in case the user introduce a 'key' that do not exist.

After we need to add 'value' corresponding to the 'key', and store it on an empty string.

In each iteration we get the result and add it to our 'output' string. The '+ '' ' in the end, represents a space so the 'one', 'two' and so on, are not close to each other.

Functions

'Functions' are use to store blocks of code. As our programs grow, we need a better way to organize our code. We need to break our code into smaller, more manageable and more maintainable chunks which we call 'functions'.

So a 'function' is a container with a few lines of code that perform a specific task. For example, we saw a few built in functions like 'print()', 'input()', and each of this functions have a purpose, they know how to perform a specific task.

So when building large complex programs, we should break up our code into more smaller and reusable chunks which we call 'functions' to better organize our code.

Let us built a program for printing a greeting message:

            def greet_user():
                print('Hi there!')
                print('Welcome aboard.')
                
            print('Start')
            greet_user()
            print('Finish')

In this code, 'def' is a reserve word in Python, and is short for define. When Python inerpreter sees this, it knows that we are defining a 'function'.

Next we need to give our 'function' a name, 'greet_user', and by using ' : ' on the end of 'def greet_user():' we are saying to Python that we are defining a block of code. Any code writen inside this 'function', belongs to this 'functions' and will be executed whenever we call this 'function'.

In the program above, the execution of our program starts in 'print('Start')', after that Python sees that we are calling the 'greet function', it will jump to the top of our program were we have define the 'greet_user' function and will execute the two lines of code inside the function, then it will jump out of this 'function' and continue the normal execution of our code, so it will print the 'Finish' message on the terminal.

To define a function remember that:

  • All the best practices seen so far regarding to naming variables, also apply to functions.
  • 'Functions' names should always be meaningful and descriptive.
  • For naming a 'functions', we should use lower case characters.
  • If the name of the 'function' has multiple words, we need to separate them using an underscore ( '_' ).
  • The order of the code matters, meaning that if we try to call a 'function' and only after we define that same 'function', we will see an error. So we always define our functions first and then call them.
Parameters

'Parameters' are used to pass information to 'functions'. Let us see how we can pass a parameter to the 'greet_user' function that we have defined:

            def greet_user(name):
                print(f'Hi {name}!')
                print('Welcome aboard.')
            
            print('Start')
            greet_user('John')
            print('Finish')

In this code, we set a parameter for our 'greet_user()' function, inside '( )'. These parameters are place holders for receiving information, in this case 'name'. With this, we can pass the name of the user, when calling this function.

In our case the parameter has set to 'John', so 'name' will act like a local variable that we defined inside of this function and that can receive information. When a function has a parameter, in this case 'name', we are obligated to pass a value for that parameter, otherwise we will get an error message saying 'missing 1 required positional argument'.

An important note is that 'parameters' and 'place holder' are not the same thing. 'Parameters' are the 'holes' or 'placeholders' that we define in our function for receiving information ('name'), and 'arguments' are the actual pieces of information that we supply to these functions ('John').

We can also define multiple 'parameters' like this:

            def greet_user(first_name, last_name):
                print(f'Hi {first_name} {last_name}!')
                print('Welcome aboard.')
            
            print('Start')
            greet_user('John', 'Smith')
            print('Finish')
Keyword Arguments

We have 'positional arguments' and 'keyword arguments'. In the first case the position of the aruments that we set in a function matter, and in 'keyword arguments' the position does not matter.

            def greet_user(first_name, last_name):
                print(f'Hi {first_name} {last_name}!')
                print('Welcome aboard.')

            print('Start')
            greet_user('John', 'Smith')
            print('Finish')

In the code above, the position in which we set the arguments that we pass to our 'greet_user' function matter. We would get different results if we set 'greet_user('John', 'Smith')' or 'greet_user('Smith', 'John')'. In the first case 'John' parameter will be set in the placeholder 'first_name' and 'Smith' parameter will be set in the placeholder 'last_name', and in the second case it would be set vice-versa. That is why these are called 'positional arguments'.

With 'keyword arguments' we can set the placeholders as we want regardless the order in each we provide them. Here is how:

            def greet_user(first_name, last_name):
                print(f'Hi {first_name} {last_name}!')
                print('Welcome aboard.')
            
            print('Start')
            greet_user(last_name='Smith', first_name='John')
            print('Finish')

When we have a parameter name ('first_name') followed by its 'value' ('John'), that is what we call a 'keyword argument'. And with this 'keyword arguments' the position does not matter, we do not have to care about the position.

Note that, most of the time, we use 'positional arguments' and not 'keyword arguments', but in certain situation, these 'keyword arguments' can help us improve the readability of our code. In the example used above, is easy to guess what the arguments represent, but sometimes we are dealing with functions and we are passing numerical values to them, in those situations it may not be quite clear what those values represent.

Let us see an example. Suppose we have a function to calculate the total cost of an order:

            def calc_cost(total, discount, shipping):
                return (total * (1 - discount) + shipping)
            
            
            total_cost = calc_cost(total=100, discount=0.1, shipping=5)
            print(f'Total cost of the order: ${total_cost}')

In this code, the first number we supply to our 'calc_cost()' function, represents the 'total' ('100'), the other is the is the 'discount' ('0.1') and the last one is the 'shipping cost' ('5').

As we see, someone reading this code may not know what these values represents, and in this case, we can improve the readability of this code, by using 'keyword arguments'. So we can prefix each argument with the name of the parameter they target, for example, we can set 'total=100', 'discount=0.1' and 'shipping=5'. Like this if someone else was reading this code, they could immediatelly tell, what these values represent. So that is the power of 'keyword arguments'.

'Keyword arguments' should always come after 'positional arguments'. If we are mixing 'positional arguments' with 'keyword arguments', we should always use 'positional arguments' first and then the 'keyword arguments'.

As final notes keep in mind:

  • For most part we should use 'positional arguments', but if we are dealing with functions that take numerical values, and is not quite clear what does values represent, we should use 'keyword arguments', to improve the readability of our code.
  • To use 'keyword arguments', simply prefix the arguments that we pass with the name of their pair parameters.
  • Finally if we are using both 'positional arguments' and 'keyword arguments', we should use 'keyword arguments' after the 'positional arguments'.
Return Statement

'Return Statements' are functions that return values. This is particularly useful, if we are doing some kind of calculation and we want to return the result, to whoever is using the function.

Let us try to build a function, that calculates the square of a number introduced by the user:

            def square(number):
                return (number * number)
            
            result = square(3)
            print(result)

In this code, first we define a function ('square') and we set the parameters ('number'). Inside the function we introduce the calculation formula ('number * number'). Now to return the value outside the function, we use the 'return' statement ('return number * number'). After we call the function and give it an argument of '3' ('square(3)'), now this functions returns a value, that we can store it in a variable ('result') and print it on the terminal.

The code above, could be rewriten in a shorter way:

            def square(number):
                return (number * number)
            
            print(square(3))

As we can see, this was a very simple program, but in real programs, we might have functions with several lines of code, to calculate something. So in does cases in particular, the 'return' statement is very valuable. The 'return' statement allow us to return values to the callers of our function.

On the code above, if we did not use the 'return' statement, like this:

            def square(number):
                print(number * number)
            
            print(square(3))

Besides having '9' (square of '3'), we would also see 'None' on the terminal.

When Python interpreter executes this code, first it will call the 'square()' function. So the control moves to the 'square' function, and there it will calculate the 'square' of the given number ('3'), and then it will print it on the terminal. That is why we see '9' on the terminal.

We will also see 'None' on the terminal, because by default, all the functions return the value 'None', if they do not have the 'return' statement inside the 'def' block of code. 'None' is an object that represents the absence of a value, it is like nothing, or 'null' in C, C++, JAVA and JAVAScript.

On the example above, we did not have a 'return' statement, so by default, Python returned the value 'None' from this function. After the 'square(3)' function is executed, the value 'None' is returned, and passed as an argument to the 'print' function ('print(square(3))'). That is the reason, why we see 'None' on the second line in the terminal.

As a final note, we should now that by default, all functions in Python return 'None' and that we can change this, by using the 'return' statement. With this we will receive the result of a function.

Reusable Functions

'Reusable functions' are functions that we use recurrently.

In 'reusable functions' we do not use the 'input()' function, because inputs can come in different froms. We do not want to put 'input()' in functions code, because they could not be reusable.

By the same principal, we should not include 'print()' functions in our 'def' function code, because what we do with the output is different from one program to another.

To define 'reusable functions' bare in mind:

  • Use 'def' and chose a name that clearly tells what the functions does.
  • Use the 'return' statement to obtain the output of a function.
  • Each function should execute one, and only one function and the task should be clearly explain in the name of the function.
  • Do not use 'input()' or 'print()' functions. In the first case, inputs can come in very different froms and in the second case, what we do with the outputs is different from one program to another.
Exception Handling

'Exceptions' or logical errors (Python Logical Errors), occur at runtime, after the program pass the syntax test. When an error occurs, or exception as we call it, Python will normally stop and generate an error message. But these exceptions can be handled, using the 'try / except' statement to deal with error in our programs. Let us try to see an example where we get the user age from the terminal:

            age = int(input('Age: '))
            print(age)

In this code, we call the 'input()' function, add a lable ('Age: ') and when the user introduce something, this function returns a string. So we need to transform this string into an integer ('int(input('Age: '))') and then store the result in a variable ('age'). After we print the result in the terminal ('print(age)').

Note that after executing the code above, and if we write a numeric value, the result printed on the terminal, will be the number we entered, but we will also see a message saying 'Process finish with exit code 0'. What this means, is that our program terminated successfully, there where no errors.

But if instead of entering a numeric value, we enter something like a string ('asd' for example), in this case, we would obtain a 'ValueError' saying 'invalid literal for int() with base 10: 'asd' '. Basically what this message is telling us, is that this string 'asd', does not contain a valid number, that can be converted to an integer.

Notice on the terminal, that there is also an exit code saying 'Process finish with exit code 1'. Exit code '1' means our program crashed. So exit code '0' always means success, and anything but '0' means crashed.

Now, as a good programmer we should anticipate this situations. We do not want to let our entire program crash, just because the user entered an invalid value/input.

So instead of letting the program crash, we should handle the situation and print a proper error message. For that, we should always see the type of error the program returns, in this case 'ValueError'.

For handle errors, Python has a constructer called 'try except':

            try:
                age = int(input('Age: '))
                print(age)
            
            except ValueError:
                print('Invalid value')

In this code, in the 'except' block, we type the type of error our program may encounter, in this case a 'ValueError', and inside, we define what should happen if our program encounters that type of error. In our case, we want to print a proper error message on the terminal.

So with this 'try / except' statement, we are telling Python to execute the 'try' block, and if it encounter an error of 'ValueError', then instead of craching the program, it will print the error message defined 'Invalid value', and the exit message will state 'Process finished with exit code 0', meaning our program completed successfully, it did not crash.

This will only be true for 'ValueError' type. To other type of errors our program will crash. So let us see another example, for handeling with another type of errors/exceptions:

            try:
                age = int(input('Age: '))
                income = 20000
                risk = income / age
                print(age)
            
            except ValueError:
                print('Invalid value')

If we run this program and if we set 'age' to '0', that represents a numerical value, our program will crash once again with the error 'ZeroDivisionError: division by zero' and this happened because we can not divide a number by '0'. On the exit code we will get 'Process finished with exit code 1'.

This happens, because we could not catch this type of error/exception 'ZeroDivisionError' with the existent 'except' block. The existing block, is only capable of catching errors of type 'ValueError'. So in situations like this, we should handle different kinds of exceptions, adding new 'except statements': 

            try:
                age = int(input('Age: '))
                income = 20000
                risk = income / age
                print(age)
            
            except ZeroDivisionError:
                print('Age can not be 0')
            
            except ValueError:
                print('Invalid value')

Now, if we set 'age' to '0', we will receive a message saying 'Age can not be 0', and the exit message will state 'Process finished with exit code 0', meaning our program completed successfully, it did not crash.

So in resume:

  • We use 'try /except' blocks to handle 'exceptions' raised in our program.
  • As a good programmer, we should always anticipate these kind of 'exceptions' and handle them properly.
Comments

'Comments' are used to add notes or comments to our programs. To insert a comment we use '#' and whatever we type after '#' will not be executed. Each line in where we need to add a 'comment', should start with '#'. Here is an example:

            # This is a comment and will not be executed
            
            print('Hello World')

In this code, and after we run our program, we will see on the terminal the 'Hello World' message. The rest of the code will not be executed.

Good uses cases for 'comments':

  • Communicate something about our code.
  • Communicate things with other developers reading our code.
  • Reminders to fix things or clear things up.
  • If we have made assumptions.

Bad uses cases for 'comments':

  • Explain what the code does, explaining what the code should explain by itself.
  • Explaining the obvious, which will make our code repetitive, and it will easily become outdated, especially if we change our code and forget to change the 'comment'.
  • To much 'comments' will make code more difficult to read and understand, it will create noise on our code.

As a final note, we should use 'comments' to explain 'whys' and 'hows' not 'whats'.

Classes

'Classes' are very important in programming and they are not specific to Python, a lot of other programming languages support the notion of 'classes'. We use 'classes' to define new types.

So far we have seen examples about basic or simple types in Python (numbers, strings and booleans), and a couple of complex types (lists and dictionaries). While these types are extremely useful, they can not always be used to model complex concepts.

For example, if we think in a 'point', it is not a boolean, is not a list, is not a dictionary, it is a different kind thing. So we can use 'classes' to define new types to model real concepts. Let us see an example, where we define a new type called 'point', and this new type is going to have methods for working with 'points':

            class Point:
                def move(self):
                print('move')
            
            def draw(self):
                print('draw')
            
            
            point1 = Point()
            point1.draw()

In this code, we start by defining a 'class', by using the 'class' keyword, and right after we give our 'class' a name ('Points'). So, to assure that the 'Dog()' or 'Cat()' class, inheritance all the 'Mammal()' methods, we write 'Dog(Mammal)' and 'Cat(Mammal)'

Note that we put 'pass' inside the 'Dog(Mammal)' and 'Cat(Mammal)', because Python does not like to have empty classes (without no code inside). It accept that, but it does not like it. Using 'pass' statement we are telling Python 'Hey, pass this line. Don't worry about it', and like that we do not have an empty class.

After all of these setps, we can create a 'dog' or 'cat' object ('dog1 = Dog()') and we can also access the methods define in the parents class ('Mammal'), by using the 'dot' notation. So in the terminal we will see 'walk'.

Note that between classes we have two line spaces. If we only had one, we should see a red underline under 'class', but it will not affetc the behavier of our program, it is just a way that Python was to complain at us.

So in this code, both 'Dog()' and 'Cat()' classes have inherit the 'walk' method from the parent class, and each of the children classes ('Dog()' and 'Cat()') have their one specific methods.

To work with 'Directories and Files' in Python 3, we can see in Python Module Index, a module called 'pathlib', which provides an 'object-oriented filesystem paths'. That basically means it provides classes, that we can use to create objects to work with different directories and files.

  • If we did not had pass an argument, a string, in this case 'ecommerce', with a file or a directory, to the 'Path()' object, this object would be refering to the current directory.

    • '*.*': all the files on a the current directory, but not all the directories.
    • '*.py': all Python files in the current directory.
    • '*.xls': all EXCEL files in the current directory.
    • Very useful to write programs to automate something. For example we can iterate over all the spreadsheets in a directory, and open them and process them.

In this code, we added 'from openpyxl.chart import BarChart, Reference'. Let us see what is going on here. In the package 'openpyxl', we have a module called 'chart', and from this module we are importing two classes: 'BarChart' and 'Reference'. Now that we have these two classes, we added a chart.

First, we need to select a range of values, in this case the values of the four column, using the 'Reference' class. In this class we need to pass two arguments, the first is the 'sheet', then we add four keyword arguments: 'min_row=2' and 'max_row=sheet.max_row' (allow us to select the cells on row '2' to '4'), and, 'min_col=4' and 'max_col=4' (only select the values of column '4'). On the end, we store this result in a variable called 'values_chart'.

After, we need to create a chart. For that, we create an instance of the 'BarChart()' class and store it on the 'chart' object. Next we call 'chart.add.data()' and pass the 'values' arguments.

Finally, we need to add the chart to our sheet, so we call 'sheet.add_chart()', pass the '(chart)' object, and tell were we want the chart to be ('e2'), this is the inicial cell were our chart will be placed.
